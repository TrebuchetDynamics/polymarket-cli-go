# One-Time Browser Setup for New Users

**Last updated:** 2026-05-08

## The Situation

Pure headless (no-browser) onboarding for **new** Polymarket deposit wallet users is currently **impossible** due to a server-side gate on Polymarket's infrastructure. This is not a bug in polygolem — it is a known limitation of the Polymarket CLOB API.

### Why It's Blocked

Polymarket's `/auth/api-key` endpoint (L1 auth) requires an EIP-712 signature to create API keys. For deposit wallets (ERC-1271 smart contracts), this signature must be validated via `isValidSignature` on-chain. However, Polymarket's L1 auth endpoint does not support ERC-1271 validation — it only accepts raw ECDSA signatures from EOAs.

This means:
- ✅ EOA-owned API keys can be created headlessly
- ❌ Deposit-wallet-owned API keys **cannot** be created headlessly
- ❌ Deposit-wallet orders **require** a deposit-wallet-owned API key (server enforces `order.owner == API_KEY.owner`)

**Therefore:** New users must complete **one browser login** to create their deposit-wallet-owned API key. After that, all operations are fully headless.

## Who Needs This?

| User Type | Needs Browser Setup? | Why |
|-----------|---------------------|-----|
| Already signed up on polymarket.com | ❌ No | You already have a deposit-wallet-owned API key |
| Fresh EOA, never used Polymarket | ✅ Yes | Need to create deposit-wallet-owned API key via browser |
| Using only EOA orders (not deposit wallet) | ❌ No | EOA-owned API key works headlessly |

---

## Security Warning: Do NOT Import Raw Private Keys into Browser Extensions

**Your `POLYMARKET_PRIVATE_KEY` controls real funds.** Importing it into a browser extension (MetaMask, Rabby, etc.) exposes it to:
- Browser malware and extensions with broad permissions
- Clipboard sniffers
- Phishing sites that mimic Polymarket
- The wallet extension's own security model

**Safer alternatives (use one of these):**

| Method | Security Level | How |
|--------|---------------|-----|
| **Hardware wallet (Ledger/Trezor)** | ⭐ Highest | Connect to MetaMask; private key never leaves the device |
| **WalletConnect + mobile wallet** | ⭐ High | Private key stays on your phone; scan QR code on Polymarket |
| **Dedicated browser profile** | ⭐ Medium | Fresh Chrome/Firefox profile, install wallet, import key, do signup, delete profile |
| **Raw key import into MetaMask** | ⭐ Low | Only for small amounts or test keys; avoid for production |

**Recommendation:** Use a hardware wallet or WalletConnect. The browser login is a one-time operation. After the API key is created, you can disconnect the wallet and never use it for polygolem again — all trading is fully headless.

## Step-by-Step Browser Setup

### Prerequisites

- A web browser (Chrome, Firefox, Safari, etc.)
- Your EOA private key already set as `POLYMARKET_PRIVATE_KEY`
- ~$0.01 POL for gas (one-time funding transfer)
- A wallet connection method (see Security Warning above)

### Step 0: Bot-Generated Keys (If Your Key Was Created by Code)

If your `POLYMARKET_PRIVATE_KEY` was generated by a bot/agent rather than imported from an existing wallet, you need a way to connect that address to a wallet for the one-time browser signup.

**Option A: Show the private key to the user (temporary import)**
```bash
# In your bot code, display the address and private key
polygolem auth status --check-deposit-key
# If depositWalletApiKeyExists is false, the user must do browser signup
```

**Option B: Generate a BIP39 mnemonic instead of raw key**
If your bot generates the key, consider generating a 12-word mnemonic instead of a raw hex key. Users can import mnemonics into any wallet (MetaMask, Rabby, Trust Wallet, etc.) and remove the account after signup.

**Option C: Display as QR code for mobile wallet import**
Some bots can display the private key as a QR code. The user scans it with MetaMask mobile, connects to Polymarket via WalletConnect, signs up, then removes the imported account.

**Security note for bot authors:** After browser signup, the private key should only exist in the bot's secure storage (environment variable, secret manager, encrypted file). Never log it to stdout or persist it to disk unencrypted.

### Step 1: Derive Your Deposit Wallet Address (Local)

Run this locally to find your deterministic deposit wallet address:

```bash
export POLYMARKET_PRIVATE_KEY="0x..."
polygolem deposit-wallet derive
```

Save the `depositWallet` address from the output. You'll need it in Step 4.

### Step 2: Get Headless Relayer Credentials

These work headlessly and are needed for deployment:

```bash
polygolem auth headless-onboard
```

This creates `../go-bot/.env.relayer-v2` with your relayer API key.

### Step 3: Deploy Your Deposit Wallet (Headless)

```bash
polygolem deposit-wallet deploy --wait
```

Wait for the transaction to confirm (~2 seconds).

### Step 4: Browser Login to Create Deposit-Wallet-Owned API Key

**This is the only step that requires a browser.**

1. Open your browser and go to **https://polymarket.com**
2. Click **"Connect Wallet"**
3. Connect the wallet that holds your EOA (the same address derived from `POLYMARKET_PRIVATE_KEY`):
   - **Hardware wallet (recommended):** Connect Ledger/Trezor to MetaMask → select your EOA address
   - **Mobile wallet (recommended):** Use WalletConnect → scan QR code with Rainbow, Trust Wallet, etc.
   - **Browser extension (only for test/small amounts):** MetaMask or Rabby — import via private key if you must
4. Polymarket will automatically:
   - Detect your existing deposit wallet (deployed in Step 3)
   - Create a deposit-wallet-owned CLOB API key in the background
5. **Verify the key was created:**
   - Go to https://polymarket.com/settings?tab=builder
   - Look for "CLOB API Keys" section
   - You should see an active key listed
6. **Disconnect or remove the wallet** after verification — you won't need it again

> **Note:** You do NOT need to copy or save this key. Polygolem will automatically derive it on-demand using your `POLYMARKET_PRIVATE_KEY`.

### Step 5: Approve and Fund (Headless)

After browser login, everything is headless again:

```bash
# Run the full approval batch
polygolem deposit-wallet approve --submit

# Fund your deposit wallet with pUSD
polygolem deposit-wallet fund --amount 0.71
```

### Step 6: Verify and Trade (Headless)

```bash
# Update CLOB balance cache
polygolem clob update-balance --asset-type collateral

# Check balance
polygolem clob balance --asset-type collateral

# Place your first order
polygolem clob create-order --token <TOKEN_ID> --side buy --price 0.5 --size 10
```

## Alternative: Quick Onboard After Browser Setup

If you've already done the browser login, you can run the full headless sequence:

```bash
export POLYMARKET_PRIVATE_KEY="0x..."

# 1. Relayer credentials (headless)
polygolem auth headless-onboard

# 2. Full onboarding — deploy + approve + fund (headless)
polygolem deposit-wallet onboard --fund-amount 0.71

# 3. Sync and verify
polygolem clob update-balance --asset-type collateral
polygolem clob balance --asset-type collateral

# 4. Trade
polygolem clob create-order --token <TOKEN_ID> --side buy --price 0.5 --size 10
```

## Verifying Your Setup

Run the built-in check to see if your deposit-wallet-owned API key is working:

```bash
polygolem auth status
```

Expected output for a working setup:
```json
{
  "eoaAddress": "0x...",
  "depositWallet": "0x...",
  "depositWalletDeployed": true,
  "eoaApiKeyExists": true,
  "depositWalletApiKeyExists": true,
  "canTrade": true
}
```

If `depositWalletApiKeyExists` is `false`, you need to complete the browser login (Step 4 above).

## Troubleshooting

### "Invalid L1 Request headers" when creating API key

This means you're trying to create a deposit-wallet-owned API key headlessly. **This is expected to fail.** Use the browser login method instead.

### "the order owner has to be the owner of the API KEY"

This means you're trying to place a deposit-wallet order with an EOA-owned API key. **This is expected to fail.** The server enforces that deposit-wallet orders must use deposit-wallet-owned API keys.

### Browser shows "Create Account" instead of detecting my deposit wallet

Make sure you deployed the deposit wallet in Step 3 before opening the browser. The wallet must exist on-chain for Polymarket to detect it.

### I already have a Polymarket account from before

If you signed up on polymarket.com before (even months ago), you likely already have a deposit-wallet-owned API key. Run `polygolem auth status` to verify. You can skip the browser setup entirely.

### I absolutely will not import my private key into any browser wallet

**Options:**

1. **Use a hardware wallet** — Connect Ledger/Trezor to MetaMask. The private key never leaves the hardware device. This is the recommended approach for production keys.

2. **Use WalletConnect with a mobile wallet** — Your private key stays on your phone. You scan a QR code on the Polymarket website and sign the login message on your mobile device.

3. **Use a dedicated browser profile** — Create a fresh Chrome/Firefox profile, install MetaMask, import the key, complete the signup, verify the API key exists, then delete the entire browser profile. The key is never in your main browser.

4. **Accept the limitation** — If none of the above work for you, pure headless onboarding is not possible with current Polymarket infrastructure. You would need to use an EOA-only trading flow (not deposit wallet) or wait for Polymarket to add ERC-1271 support to their L1 auth endpoint.
