Polygolem is a Go protocol and automation stack for Polymarket with a thin
Cobra-based CLI on top. The codebase is split into three layers:

- **`cmd/polygolem`** — the binary entry point. Just wires Cobra and exits.
- **`internal/` (21 packages)** — implementation: HTTP clients, signers,
  config, paper executor, modes, output, errors. Not part of the public API.
- **`pkg/` (5 packages)** — the stable public Go SDK: `bookreader`, `bridge`,
  `gamma`, `marketresolver`, `pagination`. Semver-stable.

Dependencies flow **inward only**: `cmd → internal → pkg`. Public SDK packages
must not import internal packages, and internal packages must not depend on
the CLI layer.

The mode boundary (read-only / paper / live) is enforced in `internal/modes`
and consumed by every command before any side-effecting code path. Live
commands additionally route through `internal/execution` after the four-gate
preflight check.
