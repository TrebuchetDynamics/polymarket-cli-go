Public packages for consumption by `go-bot` and other Go projects. This page
records the current exported contract: packages, methods, and the exact types
in their signatures.

The `internal/` packages are implementation detail. They are documented in
[Internal Packages](/reference/internal-packages), but they do not carry the
same semver promise as the `pkg/` packages below.

Some current `pkg/` method signatures still expose implementation types from
`internal/polytypes`, `internal/clob`, `internal/dataapi`, or `internal/stream`.
That is an SDK hygiene gap, not an upstream Polymarket requirement. The method
surface is documented here so it is visible until those types are promoted or
re-exported under public package paths.

## Known Public SDK Gaps

The following are intentional gaps in the current stable `pkg/` contract:

| Gap | Current Status |
|---|---|
| One-call deposit-wallet onboard SDK | Relayer primitives are public in `pkg/relayer`; full onboard orchestration remains CLI-only |
| Public type ownership | Several signatures still reference `internal/*` implementation types; promote or re-export them before claiming a clean external SDK |
| Non-zero V2 `builderCode` attribution | The signer emits `builder = bytes32(0)` today |
| Batch order placement | Single-order placement and cancellation are exposed; batch placement is not |
| Authenticated user WebSocket stream | Public market stream is exposed; user stream waits for L2 WebSocket auth tests |
| Consolidated V2 CLOB market info | Tick size, neg-risk, and fee-rate are separate calls today |
