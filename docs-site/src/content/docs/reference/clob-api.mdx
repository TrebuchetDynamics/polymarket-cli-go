The CLOB API (`https://clob.polymarket.com`) is Polymarket's order book and
trading layer. It has two tiers:

- **L0 / read-only** ã order book depth, midpoint, spread, tick size, fee rate,
  last trade. No authentication required.
- **L1 / L2 / authenticated** ã create order, cancel, query, trades, rewards,
  scoring, balances. Requires an EOA private key + deposit wallet.

Polygolem implements the full surface in `internal/clob`. Read-only access is
also exposed via the public `pkg/bookreader` package.

## Read-only commands

```bash
polygolem orderbook get        --token-id 71321045679...
polygolem orderbook price      --token-id 71321045679...
polygolem orderbook midpoint   --token-id 71321045679...
polygolem orderbook spread     --token-id 71321045679...
polygolem orderbook tick-size  --token-id 71321045679...
polygolem orderbook fee-rate   --token-id 71321045679...
polygolem orderbook last-trade --token-id 71321045679...
```

See [Orderbook Data](/guides/orderbook-data).

## Authenticated commands

```bash
polygolem clob create-order --signature-type deposit ...
polygolem clob market-order --signature-type deposit ...
polygolem clob orders
polygolem clob order <order-id>
polygolem clob trades
polygolem clob cancel <order-id>
polygolem clob cancel-orders <order-id-1,order-id-2>
polygolem clob cancel-market --market <condition-id>
polygolem clob cancel-all
polygolem clob update-balance --asset-type collateral --signature-type deposit
```

Read-only CLOB market discovery is also available through:

```bash
polygolem clob markets --cursor ""
polygolem clob market <condition-id>
polygolem clob price-history <token-id> --interval 1m
```

Polygolem supports **deposit wallet (type 3 / POLY_1271)** exclusively.
EOA, proxy, and Gnosis Safe are blocked by CLOB V2 for new API users
and are not supported in this CLI.

| Type | CLI alias | Status |
|------|-----------|--------|
| 0 | `eoa` | ã Blocked by CLOB V2 |
| 1 | `proxy` | ã Blocked by CLOB V2 |
| 2 | `safe` / `gnosis-safe` | ã Blocked by CLOB V2 |
| 3 | `deposit` / `deposit-wallet` / `poly-1271` | ã Required |

Polygolem's CLOB commands default to `--signature-type deposit`.

Cancel commands require L2 credentials because they mutate upstream order
state, but they remove open exposure instead of creating it. Order placement
requires the deposit wallet path.

<Aside type="caution">
**Deposit wallet is mandatory.** Polymarket CLOB V2 requires deposit wallet
(POLY_1271) for order placement. See
[Deposit Wallets & Signature Types](/concepts/deposit-wallets) for the full
onboarding flow.
</Aside>
