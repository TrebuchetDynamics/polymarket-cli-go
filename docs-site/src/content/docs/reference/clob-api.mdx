The CLOB API (`https://clob.polymarket.com`) is Polymarket's order book and
trading layer. It has two tiers:

- **L0 / read-only** — order book depth, midpoint, spread, tick size, fee rate,
  last trade, price history. No authentication required.
- **L1 / L2 / authenticated** — create order, cancel, list orders/trades, rewards,
  scoring, balances. Requires EOA private key + L2 API credentials.

Polygolem implements the full surface in `internal/clob`. Read-only access is
also exposed via `pkg/bookreader` and `pkg/universal`.

## Read-only commands

```bash
polygolem orderbook get        --token-id <token-id>
polygolem orderbook price      --token-id <token-id>
polygolem orderbook midpoint   --token-id <token-id>
polygolem orderbook spread     --token-id <token-id>
polygolem orderbook tick-size  --token-id <token-id>
polygolem orderbook fee-rate   --token-id <token-id>
polygolem orderbook last-trade --token-id <token-id>
polygolem clob book <token-id>
polygolem clob tick-size <token-id>
polygolem clob price-history <token-id> --interval 1m
polygolem clob markets --cursor ""
polygolem clob market <condition-id>
```

See [Orderbook Data](/guides/orderbook-data).

## Authenticated commands

```bash
# Order management
polygolem clob create-order   --token <id> --side buy --price 0.5 --size 10 --signature-type deposit
polygolem clob market-order   --token <id> --side buy --amount 5 --signature-type deposit
polygolem clob cancel         <order-id>
polygolem clob cancel-orders  <order-id-1,order-id-2>
polygolem clob cancel-market  --market <condition-id>
polygolem clob cancel-all

# Account queries
polygolem clob orders
polygolem clob order          <order-id>
polygolem clob trades
polygolem clob balance        --asset-type collateral --signature-type deposit
polygolem clob update-balance --asset-type collateral --signature-type deposit

# API key management
polygolem clob create-api-key
```

Polygolem supports **deposit wallet (type 3 / POLY_1271)** exclusively.
EOA, proxy, and Gnosis Safe are blocked by CLOB V2 for new API users.

| Type | CLI alias | Status |
|------|-----------|--------|
| 0 | `eoa` | Blocked by CLOB V2 |
| 1 | `proxy` | Blocked by CLOB V2 |
| 2 | `safe` / `gnosis-safe` | Blocked by CLOB V2 |
| 3 | `deposit` / `deposit-wallet` / `poly-1271` | Required |

## Authentication Flow

CLOB V2 uses a two-level auth system:

1. **L1** — EOA signs an EIP-712 `ClobAuth` message, posted to `POST /auth/api-key`
2. **L2** — CLOB returns `{ apiKey, secret, passphrase }`, used for HMAC-SHA256 request signing
3. **POLY_1271** — orders signed by the EOA, validated by the deposit wallet via ERC-1271

```bash
# One command: create builder profile + L2 creds
polygolem builder auto
# Writes BUILDER_API_KEY, BUILDER_SECRET, BUILDER_PASSPHRASE to env file
```

The same HMAC credentials work for both CLOB L2 endpoints AND relayer v2 endpoints.

## V2 Order Envelope

Orders in CLOB V2 use the `Polymarket CTF Exchange` domain (version `"2"`) and include:
- `salt`, `maker`, `signer`, `tokenId`, `makerAmount`, `takerAmount`
- `side` (0=BUY, 1=SELL), `signatureType` (3=POLY_1271)
- `timestamp`, `metadata`, `builder` (bytes32 for builder fee attribution)

For POLY_1271, the order signature is wrapped in an ERC-7739 `TypedDataSign` envelope:
`innerSig(65) || appDomainSep(32) || contents(32) || contentsType(186) || uint16BE(186)` = 636 hex chars.

See [`docs/JSON-CONTRACT.md`](https://github.com/TrebuchetDynamics/polygolem/blob/main/docs/JSON-CONTRACT.md) for the full wire format.
