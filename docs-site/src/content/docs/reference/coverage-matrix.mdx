## Matrix

| Surface | SDK | CLI | Status |
|---|---|---|---|

| Gamma taxonomy | `internal/gamma`, `pkg/universal` | `discover tags`, `discover series`, `discover comments` | Covered |
| CLOB public data | `internal/clob`, `pkg/bookreader`, `pkg/universal` | `orderbook *`, `clob book`, `clob market`, `clob markets`, `clob price-history` | Covered |
| CLOB account reads | `internal/clob`, `pkg/universal` | `clob create-api-key`, `clob balance`, `clob update-balance`, `clob orders`, `clob order`, `clob trades` | Covered |

The repository copy with test and documentation columns lives in
[`docs/POLYMARKET-COVERAGE-MATRIX.md`](https://github.com/TrebuchetDynamics/polygolem/blob/main/docs/POLYMARKET-COVERAGE-MATRIX.md).

## Gaps

- Authenticated user WebSocket streams wait for L2 WebSocket auth tests.
- Configurable V2 `builderCode` attribution is not yet a CLI or public SDK contract.
- Batch order placement is not yet exposed.
- Data API all-market open interest is not exposed by the CLI yet; the current
  command requires `--token-id`.
- The command reference is manually aligned with the CLI tree. A checked-in
  generator is still a documentation tooling gap.
