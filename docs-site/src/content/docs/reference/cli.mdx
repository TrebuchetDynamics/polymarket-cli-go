<Aside type="caution">
Trading commands always use deposit-wallet signing (type 3 / POLY_1271).
The old `--signature-type` flag has been removed.
</Aside>

## Discovery

```bash
polygolem discover search --query "btc 5m" --limit 10
polygolem discover markets --limit 20 --active
polygolem discover market --id "0xbd31dc8a..."
polygolem discover market --slug "will-btc-be-above"
polygolem discover enrich --id "0xbd31dc8a..."
polygolem discover tags --limit 100
polygolem discover series --limit 20
polygolem discover comments --entity-id 123 --entity-type market
```

Discovery commands are read-only and require no credentials.

## Credentials

```bash
POLYMARKET_PRIVATE_KEY="0x..." polygolem builder auto
POLYMARKET_PRIVATE_KEY="0x..." polygolem auth headless-onboard
POLYMARKET_PRIVATE_KEY="0x..." polygolem clob create-builder-fee-key
```

`builder auto` creates CLOB L2 credentials. `auth headless-onboard` mints the
V2 relayer key used by deposit-wallet deploy and approval batches. Builder fee
keys are CLOB-side attribution keys; use the returned key as
`POLYMARKET_BUILDER_CODE` or pass it to `--builder-code`.

## CLOB

```bash
polygolem clob markets --cursor ""
polygolem clob market <condition-id>
polygolem clob orders
polygolem clob order <order-id>
polygolem clob trades
polygolem clob create-api-key-for-address --owner <deposit-wallet>
polygolem clob create-builder-fee-key
polygolem clob list-builder-fee-keys
polygolem clob cancel <order-id>
polygolem clob cancel-orders <order-id-1,order-id-2>
polygolem clob cancel-market --market <condition-id>
polygolem clob cancel-all
```

Authenticated CLOB account commands require `POLYMARKET_PRIVATE_KEY` so
polygolem can derive L2 credentials. Cancel commands mutate upstream order
state but reduce existing exposure; order placement still requires deposit
wallet signing. V2 builder attribution is a bytes32 order field: pass
`--builder-code` explicitly or set `POLYMARKET_BUILDER_CODE` (alias:
`POLYMARKET_CLOB_BUILDER_CODE`).

```bash
polygolem clob create-order \
  --token <token-id> --side buy --price 0.5 --size 10 \
  --builder-code "$POLYMARKET_BUILDER_CODE"

polygolem clob market-order \
  --token <token-id> --side buy --amount 5 \
  --builder-code "$POLYMARKET_BUILDER_CODE"
```

```bash
polygolem data trades --user 0x... --limit 20

polygolem data holders --token-id <token-id> --limit 20

```

## Streams

```bash
polygolem stream market --asset-ids <token-id-1>,<token-id-2> --max-messages 10
```

The CLI exposes the public market WebSocket stream. The authenticated user
stream remains SDK-planned until L2 WebSocket auth has local tests.
